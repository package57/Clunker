# Clunker
  The idea behind Clunker is to de mystify C++, far beyond a Reference or Tutorial. The C++ language although powerful, is a tough nut to crunch.
The reference seem clear enough, but look beyond that at any project and things to outerspace fast. I intend to wrangle C++. 
 
  In or 40 years exprience programming, building and deploying mission critical applications, I've never created 'an Object'. I've heard that there
is nothing you can do in "C++" that you can't do in "C". So, why would you use C++ and leverage it's Object Oriented features? We'll get to that.

  Another thing that's glaring is the lack of file processing knowledge. The References perhaps have a single page dedicated to files. I find this
appaling. Information Technology and Data Processing have two things in common: files. The Tutorials are fucused on little bites of data being passed 
around. My last project I was dealing with multiple Peta bytes of data. I'll un shroud that, and cover Databases too. 

  Let's I pull the viel off presentation. In my life I never used a Graphical User Interface aka "GUI". All I ever did was manipulate characters
so a Character User Interface or "CUI" was more than adequate. So what's a GUI for? We'll see.

  The biggest 'miss' in References and Tutorials is the "Why" of everything. I beleieve that time will be better spent learning why use a feature.
After that "syntax is syntac" right? Well, with C++ you'll spend lot's of time figuring out syntax. it's it's own little monster. There are no doubt 
volumes written about C++ syntax, and why it's structured as such. 

  If you are comming to this as a complete novice, that's ok. I'll make connections to the 'read.me' with the code. 
  
  Let's begin, shall we. 
  
What are we doing again?

  If you do something once, you certainly don't need a computer. You might not even use a tool (IDE). If you do anything more than once, that get's interesing.
You will probably want to remember how you did it, write down the steps. Make a note of what actions belong to what result. Want to make a cake? Once
you decide what kind (condition), it will be made with specified ingredients (data), combined and acted upon (logic) in a specific order (sequence). 
You'll probably want to share that with someone else in the future (class). Now, go out and make a million cakes a day (iteration).

  We need a tool, a machine, to do the work for us. We need to have a way to tell the computer what we want done. There are thousands of computer 
languages, new ones come along almost daily. Many exist to faciltate a specific type of work. I could go on and on here, but I wont. Let's use C++
as our language of chice. 

  So what we are doing is converting a logical unit of work, into instructions that comply with C++, so that we can go on with our busy lives.  

C++ is funny.
